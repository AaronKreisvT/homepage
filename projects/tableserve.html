TableServe

Lightweight web app for table-specific orders via QR codes. No payment, just ordering.

Overview

  • Stack: HTML, CSS, JavaScript (vanilla), PHP 8 (Apache + PHP-FPM), CSV files (no DB), QR codes, jsPDF (PDF export)
  • Status: Finished; iteratively refining styles and content
  • Repo: GitHub

Notes

  1. Start: Site currently running at tableserve.kreisaaron.de. Codes can be found on GitHub.
  2. Guests: Scan the QR code at the table ⇒ order.html?code=XXXX. Select drinks, optionally add a comment, then submit the order.
  3. Staff/Bar: Open /staff.html, log in with the staff key. View open orders, update status (in prep/served/cancelled), print receipts, generate daily PDF reports.
  4. Admin: Open /admin.html with the admin key. Manage tables (name/code) and menu (items/price/category/active).
  5. Data storage: CSV files in /data (protected from direct access via .htaccess). No database required.
  6. Order IDs: Table code prefix + random suffix (collision-safe).
  7. Printing: Receipts via browser print (80 mm thermal), PDF reports via jsPDF.
  8. Setup: Upload files, adjust config.php (keys/paths), enable PHP-FPM & SSL — done.
